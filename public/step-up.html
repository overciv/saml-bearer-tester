Step 1 — Configuration
Step-Up Authentication — request a higher assurance level by including acr_values in the authorization request. Okta challenges the user with additional factors and issues a token with the acr claim set to the satisfied level.
Step 2 — Baseline Token (standard auth)
Get a standard token first (no step-up). This will show the initial low-assurance acr value for comparison. Optional — skip if you want to jump straight to step-up.
Step 3 — Step-Up Request
A popup will open to Okta with the acr_values parameter. Okta prompts the user to satisfy the required assurance level (e.g. MFA). The resulting token will have acr set to the satisfied level.
Step 4 — Token Comparison
Complete Steps 2 and 3 to see the before/after comparison here.
