Step 1 — Configuration
Auth Code + PKCE — human authentication. The user logs in via a browser popup. PKCE is generated automatically (no client secret needed for public clients).
Must be registered in the Okta app
Web apps typically use Basic or PKJWT
Step 2 — Authorization
A popup will open to Okta. The user logs in. PKCE verifier is generated and stored server-side. The popup closes automatically and tokens appear below.
Step 3 — Tokens
Complete authorization to see tokens here.
