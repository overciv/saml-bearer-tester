Single Logout (OIDC RP-Initiated)
Redirects the browser to Okta's /v1/logout endpoint via GET (canonical OIDC RP-Initiated Logout). Okta terminates the SSO session, then — for each app that has SLO enabled — embeds a hidden iframe loading that app's frontchannel_logout_uri. Only after all iframes have fired does Okta redirect back to the app.
⚠️ For cross-app SLO to propagate, each participating OIDC app must be configured:
  1. Set participateSlo: true in settings.oauthClient (via PUT /api/v1/apps/{id})
  2. Set frontchannel_logout_uri to a URL in that app that invalidates the session
  3. Optionally set frontchannel_logout_session_required: true so Okta appends sid+iss to scope the logout to a specific session
Okta uses hidden iframes to POST to each app's frontchannel_logout_uri before redirecting back. Check user.session.end in the System Log — the TotalOidcLogoutRequests debug field shows how many apps received the notification.
