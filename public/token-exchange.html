Step 1 — App B Configuration (the requesting app)
RFC 8693 Token Exchange — App B authenticates with its own credentials and presents App A's token as the subject_token. Okta validates App A's token, applies App B's access policies, and issues a new token with App B's cid. The user identity (sub) is preserved.

App B must have urn:ietf:params:oauth:grant-type:token-exchange in its grant types.
The token issued will have this as cid
