Step 3 — Token Polling
Poll Okta's /token endpoint with grant_type=urn:openid:params:grant-type:ciba every 5s (from the interval field in the authorize response). Keep polling while Okta returns authorization_pending. Stop on slow_down (increases interval by 5s), access_denied, expired_token, or success.

Okta Setup Guide
⚠️ CIBA requires a Custom Authenticator — a branded mobile app built with the Okta Devices SDK. There is no way to test CIBA without an enrolled Custom Authenticator that receives push notifications.
1. Configure the Okta App
  1. Apps → your app → General → Grant types
  2. Enable urn:openid:params:grant-type:ciba
  3. Set Preferred authenticator for CIBA → your Custom Authenticator
  4. Optionally set backchannel_token_delivery_mode = poll (only supported mode)
2. Enable in Auth Server Policy
  1. Security → API → your auth server → Access Policies
  2. Edit the default policy rule → Grant types → Advanced
  3. Enable Client-initiated backchannel authentication (CIBA)
3. Custom Authenticator Enrollment
  1. Build a mobile app using the Okta Devices SDK (Swift / Kotlin)
  2. Enroll the user's device via enrollAuthenticator()
  3. Enable "Sign in with push notifications" in the authenticator app
  4. Enable CIBA transactions in the push notification settings
